üéØ Icon Button Fix - Manual Test

This page provides test cases to verify icon buttons work correctly in AkselArcade.

üìã Test Instructions

  1. Open http://localhost:5173 in a new tab
  2. Copy one of the test cases below
  3. Paste it into the editor
  4. Verify the button renders with the icon in the preview pane
  5. Open browser console (F12) and check for the success messages

Test Case 1: Icon Only Button

A button with only an icon, no text.

<Button icon={<PencilIcon title="Rediger" />} />

Expected: A button with a pencil icon

Test Case 2: Icon + Text (Left Position)

A button with an icon on the left and text.

<Button icon={<PencilIcon aria-hidden />}>Rediger</Button>

Expected: A button with pencil icon on the left and "Rediger" text

Test Case 3: Icon + Text (Right Position)

A button with an icon on the right and text.

<Button iconPosition="right" icon={<PencilIcon aria-hidden />}>Next</Button>

Expected: A button with "Next" text and pencil icon on the right

Test Case 4: Styled Icon Button

A button with variant styling and icon.

<Button variant="primary" icon={<PencilIcon aria-hidden />}>Edit Document</Button>

Expected: A primary-styled button with pencil icon and "Edit Document" text

Test Case 5: Multiple Icon Buttons

Multiple buttons with different icons.

<div style={{ display: 'flex', gap: '1rem' }}>
  <Button icon={<PencilIcon aria-hidden />}>Edit</Button>
  <Button icon={<TrashIcon aria-hidden />}>Delete</Button>
  <Button icon={<PlusIcon aria-hidden />}>Add</Button>
</div>

Expected: Three buttons in a row with different icons
